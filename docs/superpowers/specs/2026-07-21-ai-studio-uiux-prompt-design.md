# AI Studio UI/UX Prompt — Design Spec

**Date:** 2026-07-21  
**Product:** Wheelpedia Indonesia (OmahBan Counter Kit + learning)  
**Goal:** One long multi-mode prompt for Google AI Studio to explore best UI/UX  
**Approach:** A — One-shot full briefing  
**Visual mood:** Ramah toko keluarga (warm, local, clear Indonesian copy)

## Locked decisions

| Decision | Choice |
|----------|--------|
| Scope | Full app (not counter-only) |
| Output mode | Multi-mode: wireframe + visual + tokens + UX checklist |
| Mood | Family tire shop — friendly, not cold SaaS, not POS chrome |
| Devices | Phone-first (~375px) + laptop; min touch 44px |
| Boundary | Not a POS: no cart, checkout, nota, cost/modal |

## Prompt architecture (sections)

1. **Role** — Senior product designer + UX for Indonesian retail staff tools  
2. **Product & boundary** — Wheelpedia vs ProjectOmahBan POS  
3. **Users & jobs** — Staff awam, counter + belajar  
4. **Mood & brand** — Toko keluarga, warm, high clarity  
5. **Screen inventory** — Mandatory screens  
6. **Multi-mode deliverables** — Wireframe → visual → tokens → checklist  
7. **Hard constraints** — Accessibility, language, anti-POS  
8. **Evaluation rubric** — How to pick “best” UI  
9. **Output format** — Ordered artifacts AI Studio must produce  

## Non-goals of this prompt work

- No code generation required (design only unless asked)  
- No inventing live POS features  

## Success criteria for the prompt

- Paste once into AI Studio → coherent redesign system  
- Counter flow still P0 in hierarchy  
- Indonesian UI copy throughout  
- Clear “not a cash register” messaging in design
